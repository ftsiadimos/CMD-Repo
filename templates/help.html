Command‑Library CLI Help

This page documents the available sub‑commands for repocomcli (the CLI client).

Tip: Use repocomcli --help for a quick reference in your terminal.
1. add

Add a new command to the repository

repocomcli add "command" [-d description] [-t tags]
  • -d — Description of what the command does
  • -t — Comma‑separated tags (e.g., "docker,linux")
2. list

List all commands with optional filtering

repocomcli list [-s search] [-n limit]
  • -s — Search term to filter commands
  • -n — Limit number of results
3. show

Show full details of a specific command

repocomcli show <id>

Displays the command with formatted output including description and tags.

4. copy

Copy a command directly to clipboard

repocomcli copy <id>

Supports xclip, xsel, wl-copy (Linux) and pbcopy (macOS).

5. export

Export all commands to a JSON file

repocomcli export backup.json

Creates a backup file that can be imported later.

6. import

Import commands from a JSON file

repocomcli import backup.json

Imports commands from an exported JSON file. Duplicates are skipped.

7. config

Manage API connection settings

repocomcli config set <host> --port <port>
repocomcli config get

Configure the API server address. Default port is 5000.

Quick Examples
# Add a Docker command with description and tags
repocomcli add "docker ps -a" -d "List all containers" -t "docker,containers"

# Search for docker-related commands
repocomcli list -s docker

# Show full details of command #42
repocomcli show 42

# Copy command #42 to clipboard
repocomcli copy 42

# Export all commands to backup
repocomcli export my_commands.json

# Import commands from file
repocomcli import my_commands.json

# Configure remote API server
repocomcli config set 192.168.1.100 --port 5001
